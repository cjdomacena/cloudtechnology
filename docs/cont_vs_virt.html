Virtualization

Virtualization allows guest operating systems (OS) to run via virtual machines on top of the host OS. The guest OS should work as if it’s using its own hardware, even though it’s not. Virtualization reduces hardware needs, but decreases overall performance because the guest OS is running on top of and simultaneously with the host machine.

Containerization

Containerization captures an application inside of a “container”. The container holds the application and only the parts of the OS that are needed for the application, isolating it from the host system. The host OS is shared by all containers, making it more efficient. Applications in development can exist in multiple versions in multiple containers without affecting each other.

For additional information on containers and virtual machines, check out Cloud Management Insider or VMware.
